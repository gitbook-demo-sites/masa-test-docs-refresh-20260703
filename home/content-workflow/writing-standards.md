# Writing standards

Good test pages should be easy to scan and easy for Ask to ground in.

## Page shape

| Part | Standard |
| --- | --- |
| Title | Use the exact concept a reader would search for. |
| Description | Summarize the outcome, not the page format. |
| Headings | Keep them short and task-oriented. |
| Tables | Use them for comparisons, limits, and checklists. |
| Links | Point to the next useful action. |

{% hint style="success" %}
Prefer one clear answer over several loosely related notes. It improves both reader comprehension and AI answer quality.
{% endhint %}

## Review pass

Before publishing, confirm the page has:

* One primary user intent.
* Stable terminology.
* At least one concrete next step.
* No stale placeholders.
